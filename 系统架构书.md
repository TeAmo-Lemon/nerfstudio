# 🌟 系统架构设计书：SemGeo-FGW Splatting
**(面向 FGW 的结构感知 3D–2D 拓扑对齐 3DGS 风格化系统)**

## 核心思想 (Core Idea)
本系统彻底摒弃了传统 3D 风格化中依赖 2D 分割掩码（如 SAM）和简单特征余弦距离匹配的范式。其核心创新在于：将 3D 场景的局部区域与 2D 风格图的局部纹理，均抽象为**带有语义特征的度量测度空间 (Metric Measure Space)**。通过引入并加速**融合格罗莫夫-瓦瑟斯坦距离 (Fused Gromov-Wasserstein, FGW)**，实现跨模态的纯结构拓扑对齐（例如：基于放射状拓扑，将树冠自动匹配给漩涡纹理）。

系统划分为两条管线：
- **管线一 (`dino_splatfacto`)：** 3DGS 语义特征蒸馏训练 + 2D 风格图纹理基元分解。产出带语义特征的 3DGS 模型和风格基元。
- **管线二 (`semgeo_splatfacto`)：** 加载管线一的输出，完成聚类分解、FGW 拓扑对齐和最终风格生成。将聚类放在训练后（而非训练中），因为高斯点数量在训练期间不断变化（densification/pruning），聚类必须在固定的高斯点集上进行。

---

## 🚀 FAISS + POT 加速策略总览

本系统在两个核心计算环节引入工业级加速库，将百万级高斯点的处理从分钟级降至秒级：

- **FAISS (Facebook AI Similarity Search):** GPU 加速的向量相似度搜索库。用于替换纯 PyTorch 实现的 K-Means 聚类、KNN 近邻搜索和最近邻分配。核心优势在于 GPU 原生索引结构（`IndexFlatL2`、`IndexIVFFlat`）和高效批处理，将 $O(N^2)$ 的暴力搜索降至近似 $O(N \log N)$。
- **POT (Python Optimal Transport):** 最优传输理论的高效 Python 实现。提供 Sinkhorn-Knopp、Fused Gromov-Wasserstein (FGW)、Unbalanced OT 等算法的 GPU 加速版本，数值稳定性远优于自实现的 Sinkhorn 迭代。

| 环节 | 原方案 | FAISS / POT 加速方案 | 预期加速比 |
|------|--------|----------------------|-----------|
| 风格基元 K-Means（管线一 阶段二） | PyTorch 迭代（50 轮） | FAISS GPU K-Means | 10–100× |
| 粗粒度预聚类（管线二 1.2） | N/A | FAISS GPU K-Means | — |
| 邻域图 KNN 检索（管线二 1.3） | chunked `torch.cdist` $O(N^2)$ | FAISS `IndexFlatL2` | 5–50× |
| 代表点分配（管线二 1.6） | `torch.cdist` brute-force | FAISS `IndexFlatL2` | 3–10× |
| FGW 跨域匹配（管线二 阶段二） | 自实现 Sinkhorn 迭代 | POT `ot.gromov.fgw` | 2–5× + 更稳定 |

> **设计原则：** FAISS 和 POT 作为可选加速后端。系统默认提供纯 PyTorch 回退路径（适用于 < 5 万高斯点）；当点数超过阈值时自动切换 FAISS/POT 加速路径。两种路径共享相同接口，确保可复现性。相关超参见附录 A 的 `use_faiss`、`use_pot`、`faiss_gpu_id`。

## 🔷 FAISS 加速引擎详解

### 索引选择策略

根据使用场景选择不同的 FAISS 索引：

| 场景 | 推荐索引 | 原因 |
|------|---------|------|
| K-Means 聚类 | `faiss.Kmeans` with GPU resources | 原生 GPU K-Means，支持百万级向量 |
| 精确 KNN（< 20万点） | `IndexFlatL2` | 暴力搜索，免训练，精度 100% |
| 近似 KNN（> 20万点） | `IndexIVFFlat` | 倒排索引，训练后搜索 $O(\sqrt{N})$ |
| 余弦相似度搜索 | `IndexFlatIP` + L2 归一化输入 | 内积等价于余弦相似度 |

### 自适应图的 FAISS 加速路径

当前实现使用 chunked `torch.cdist` 构建邻域图（复杂度 $O(N^2)$）。FAISS 加速路径的流程为：

1. **构建索引：** `faiss.IndexFlatL2` 对所有高斯点坐标建立索引（L2 距离）。
2. **批量 KNN 检索：** `index.search(x, k_max)` 一次性获取每个点的 `k_max` 个最近邻及距离。
3. **特征门控后过滤：** 在 GPU 上以向量化方式检查 DINO 余弦相似度门控（`semantic_gate`），剔除跨语义邻居。
4. **自适应半径裁剪：** 仅保留满足 $d(i,j) < \alpha \cdot d_i^{(k_{base})}$ 的边。

此路径将邻域图的构建时间从 $O(N^2)$ 降至 $O(N \log N)$，且避免了 chunked cdist 的多次显存分配。

## 🔷 POT 加速引擎详解

### FGW 求解器选择

POT 提供了多种 FGW 变体，本系统根据场景选择：

| 求解器 | 适用场景 | API |
|--------|---------|-----|
| `fused_gromov_wasserstein` | 标准 FGW，有熵正则化 | `ot.gromov.fused_gromov_wasserstein(M, C1, C2, p, q, loss_fun='square_loss', alpha=0.5, epsilon=0.01)` |
| `entropic_fused_gromov_wasserstein` | 大规模点集，需要强正则化 | `ot.gromov.entropic_fused_gromov_wasserstein(...)` |
| `fgw_barycenters` | 多风格基元融合到一个 3D 区域 | `ot.gromov.fgw_barycenters(...)` |

### Coarse-to-Fine FGW 匹配流程

对于百万级高斯的场景，直接对全量代表点做 FGW 仍可能显存溢出。POT 配合由粗到细策略的完整流程：

1. **Super-node 聚合：** 在粗聚类层（管线二 1.2）上构建粗粒度度量空间 $(X_{coarse}, d_X^{coarse}, \mu_X^{coarse}, F_{coarse}^X)$。
2. **粗匹配：** 用 POT `fused_gromov_wasserstein` 求解 Super-nodes 与风格基元之间的最优传输计划 $\pi_{coarse}$。
3. **Top-K 筛选：** 对每个 Super-node，取 $\pi_{coarse}$ 中权重最高的 $K_{top}=3$ 个候选基元。
4. **细匹配：** 在 Super-node 内部的原代表点上，仅对 Top-K 候选基元做细粒度 FGW，得到最终点级匹配。
5. **组装：** 将所有细匹配结果合并，得到完整的 3D→2D 对应关系。

```python
# 伪代码示例
import ot

# Step 2: Coarse FGW
pi_coarse = ot.gromov.fused_gromov_wasserstein(
    M=1 - cos(F_coarse_X, F_Y),      # 特征代价 (交叉域)
    C1=d_X_coarse,                    # 3D 粗结构矩阵
    C2=d_Y,                           # 2D 基元结构矩阵
    p=mu_X_coarse,                    # 3D 概率测度
    q=mu_Y,                           # 2D 概率测度
    alpha=0.5,                         # 特征 vs 结构平衡
    epsilon=0.01,                      # 熵正则化系数
)

# Step 4: Fine-grained FGW per super-node
for super_label, top_k_candidates in coarse_assignments:
    pi_fine = ot.gromov.fused_gromov_wasserstein(
        M=1 - cos(F_X_fine[super_label], F_Y[top_k_candidates]),
        C1=d_X_fine[super_label],
        C2=d_Y[top_k_candidates],
        p=mu_X_fine[super_label],
        q=mu_Y[top_k_candidates],
        alpha=0.5,
        epsilon=0.005,
    )
```

# 🔵 管线一：DinoSplatfacto — 3D 特征蒸馏训练与风格基元分解
**目标：** 在 3DGS 重建的同时蒸馏 DINO 语义特征到每个高斯点；对 2D 风格图做纹理基元分解。产出带语义的 3DGS checkpoint 和风格基元 prototypes。

## 🔷 阶段一：3D 混合特征蒸馏训练

### 1.1 DINOv2 2D 特征预处理（训练前离线阶段）

在开始 3D 训练之前，先对训练数据集进行特征预提取和降维。

1. **特征提取：** 使用预训练 DINOv2（推荐 `dinov2_vitl14`）对所有训练视角图像提取 Feature Map，得到每张图的特征图 $F_{2D} \in \mathbb{R}^{H \times W \times 1024}$。

2. **全局 PCA 降维（关键）：**
   - DINOv2 原始特征维度（1024）过高，直接存入高斯点会导致显存爆炸且渲染极慢。
   - **做法：** 收集所有训练图像的特征向量（采样或全量），运行一次全局 PCA，将维度从 1024 压缩至 **16 维**。
   - **依据：** 8–16 维足以保留 DINOv2 的核心语义信息（主成分累积方差 > 80%），同时极大加速后续聚类和渲染。
   - **保存：** 将降维后的特征图 $F_{2D}^{low} \in \mathbb{R}^{H \times W \times 16}$ 保存到 `dino_features.pt`，作为后续 3D 蒸馏的真值 (Ground Truth)。

### 1.2 3DGS 模型扩展

修改 3DGS 底层数据结构，让每个高斯点同时携带颜色和语义信息。

1. **参数扩展：** 在 GaussianModel 中为每个高斯点增加属性向量 $f_{dino} \in \mathbb{R}^{16}$，形状为 `[N, 16]`。

2. **特征渲染：**
   - **原理：** 与颜色渲染一致，使用 Alpha Blending 对特征进行加权合成：
     $$F_{feat}^{render} = \sum_{i \in N} f_{dino,i} \cdot \alpha_i \prod_{j=1}^{i-1} (1 - \alpha_j)$$
   - **实现方案：**
     1. *(当前实现)* 复用 gsplat rasterizer 的多通道渲染能力（支持渲染任意 attribute），将 `dino_features` 作为 "color" 通道传入，以纯 RGB 模式渲染；
     2. *(备选)* 若需极致性能，可修改 CUDA forward/backward kernel，增加特征渲染通道。

3. **Densify/Prune 同步：** 使用自定义 `DinoDefaultStrategy`，在高斯点 clone/split/cull 时自动同步 `dino_features` 参数（gsplat 的 `duplicate`/`split`/`remove` ops 原生支持额外参数组）。

### 1.3 特征蒸馏训练

在基础 3DGS 重建训练的同时进行特征蒸馏。

- **联合损失函数：**
  $$Loss_{total} = Loss_{rgb} + \lambda_{feat} \cdot Loss_{feat}$$
  $$Loss_{feat} = \| F_{feat}^{render} - F_{2D}^{low} \|_1$$

- **延迟启用策略：** DINO 损失在训练的前 `dino_loss_start_step`（建议 3000）步不生效，让几何先行收敛，防止早期特征梯度干扰位置优化。

- **超参设置：** $\lambda_{feat}$ 建议设为 **0.01 ~ 0.1**，默认 0.05。

- **结果：** 训练完成后，每个高斯点携带一个代表其语义类别的 16 维编码 $f_{dino,i}$。输出为标准的 3DGS checkpoint（`step-*.ckpt`）。

## 🔷 阶段二：2D 风格图纹理基元分解
**目标：** 将单张风格图无监督地分解为一组局部纹理基元 prototype 向量，供管线二做 FGW 匹配使用。

1. **Patch 级特征提取：** 对风格图提取 DINOv2 patch-level 特征（使用 `dinov2_vitl14` 的 `x_norm_patchtokens`），将每个 Patch 视为特征空间中的一个点。

2. **L2 归一化与 FAISS GPU K-Means 聚类：**
   - 对所有 patch 特征做 L2 归一化。
   - 使用 FAISS GPU K-Means（`faiss.Kmeans`，`gpu=True`）将 L2 归一化后的 patch 特征聚类为 K 个 prototype（建议 $K = 5 \sim 8$）。FAISS 的 GPU K-Means 相比纯 PyTorch 迭代实现快 **10–100 倍**；对于 256×256 patch 网格（65,536 个点），聚类可在亚秒级完成。
   - 对聚类中心再做 L2 归一化，得到最终 prototype。

3. **输出：** 保存 K 个 L2 归一化的 prototype 向量到 `style_primitives.pt`，形状 `(K, D)`，其中 $D$ 为 DINOv2 特征维度（ViT-L14 为 1024）。

4. **加载：** `DinoSplatfactoModel` 在 `populate_modules()` 中自动将 style primitives 注册为 buffer，供后续使用。

> **注：** 此阶段被放在管线一而非管线二，因为它和 DINO 特征预提取一样属于"训练前预处理"——风格图的 decomposition 只依赖 2D 图像本身，不依赖 3DGS 训练结果。

# 🟢 管线二：SemGeoSplatfacto — 结构分解与 FGW 风格化
**目标：** 加载管线一的 3DGS checkpoint 和风格基元，对固定高斯点集进行聚类、构建度量空间、FGW 对齐和最终风格生成。

> **设计原因：** 管线一的聚类操作（1.4–1.9）不能在训练过程中执行，因为：
> 1. 高斯点数量在 densification/pruning 阶段不断变化，聚类结果会立即失效；
> 2. 聚类依赖全局图构建和区域生长，计算量大，不适合每个 training step 执行；
> 3. 聚类结果本身不参与梯度优化，属于后处理/推理阶段。
>
> 因此，聚类 + FGW 匹配 + 风格生成统一放在管线二中，以训练好的固定高斯点集为输入。

## 🔷 阶段一：3D 结构感知分解与度量空间构建

加载管线一产出的 checkpoint，从固定高斯点集出发，完成语义聚类和度量空间构建。

### 1.1 构建增强混合特征空间 $V_{enhanced}$

将几何特征与语义特征融合，构建用于聚类的增强特征向量。

1. **几何分量 $g_i$：**
   $$g_i = [x, y, z,\; n_x, n_y, n_z,\; SH_0]$$
   其中 $[x,y,z]$ 为空间坐标，$[n_x,n_y,n_z]$ 为法线方向（从协方差矩阵最小特征向量提取），$SH_0$ 为基础颜色（球谐直流分量）。

2. **语义分量 $f_{dino,i}$：** 蒸馏后的 16 维 DINO 特征（从 checkpoint 中读取）。

3. **标准化与白化：**
   - 对几何分量和语义分量分别进行 Z-score 标准化：$\text{Norm}(x) = (x - \mu) / \sigma$。
   - 对拼接后的向量进行 **白化 (Whitening)** 处理，消除特征维度间的相关性，平衡几何与语义在距离度量中的贡献：
     $$V_i = \text{Whiten}([w_g \cdot \text{Norm}(g_i),\; w_d \cdot \text{Norm}(f_{dino,i})])$$

4. **加权策略：**
   - 若希望分割更贴合物理边界（如区分相邻小物体），增大几何权重 $w_g$（推荐 $w_g = 1.0$）。
   - 若希望跨空间聚合同类语义物体，增大语义权重 $w_d$（推荐 $w_d = 0.5 \sim 1.0$）。
   - 默认均衡配置：$w_g = 1.0,\; w_d = 0.8$。

### 1.2 粗粒度预聚类（FAISS GPU 加速）

当高斯点数达到十万级以上时，直接构建全量邻域图代价过高。先用 FAISS 进行粗聚类可大幅降低后续图计算的规模。

1. **FAISS GPU K-Means：**
   - 输入所有有效高斯点的 $V_{enhanced}$（来自 1.1 节的增强特征）。
   - 设定粗聚类数 $K = 50 \sim 100$（有意过分割，后续可通过图生长合并相邻区域）。
   - FAISS 的 `faiss.Kmeans(d, k, gpu=True)` 将百万级点云聚类在秒级完成。

   ```python
   # 伪代码
   import faiss
   d = V_enhanced.shape[-1]  # 特征维度
   kmeans = faiss.Kmeans(d=d, k=K, gpu=True, niter=20, nredo=3)
   kmeans.train(V_enhanced.cpu().numpy())  # FAISS 自动管理 GPU 传输
   _, coarse_labels = kmeans.assign(V_enhanced.cpu().numpy())
   ```

2. **构建簇内 FAISS 索引：**
   - 对每个粗簇 $C_k^{coarse}$，构建局部的 `faiss.IndexFlatL2` 索引（仅含簇内高斯点的空间坐标）。
   - 后续 1.3 节的邻域图 KNN 检索将在各粗簇内独立执行，将复杂度从 $O(N^2)$ 降至 $O(\sum_k N_k^2) \ll O(N^2)$。
   - 若粗簇仍需进一步划分，可对超大簇（$N_k > 50,000$）启用 `IndexIVFFlat` 近似索引以进一步加速。

3. **跨簇边界处理：**
   - 粗簇边界附近的点可能被误分到不同簇，导致最终分割在边界处断裂。
   - **解决方案：** 对每个粗簇，在其边界点（距簇心距离前 10% 的点）额外检索相邻粗簇中的 KNN，确保跨簇边不被遗漏。

4. **输出：** 每个高斯点的粗聚类标签 $L_i^{coarse}$，各粗簇的 FAISS 局部索引，以及跨簇边界连接候选。

> **注：** 若场景点数较少（< 5 万），可跳过此步骤，直接进入 1.3 的全量图构建。

### 1.3 自适应邻域图构建

在粗簇内部（或全量点集上），构建保留局部拓扑结构的邻域图。

1. **有效点筛选与采样：**
   - 按不透明度阈值筛选：$\sigma(\alpha) \ge 0.05$（过滤低贡献高斯点）。
   - 若点数仍过大，按不透明度降序保留前 `max_sample_points` 个点（建议 `max_sample_points = 200,000`）。

2. **局部尺度估计（非固定 KNN）：**
   - 对每个点 $i$，计算其到第 $k_{base}$ 近邻的距离作为局部尺度：
     $$s_i = d_i^{(k_{base})},\quad k_{base} = 8$$

3. **自适应半径邻域：**
   - 定义自适应半径 $r_i = \alpha \cdot s_i$，$\alpha = 2.0$。
   - 先取最多 $k_{max} = \max(knn\_k, 32)$ 个候选近邻。
   - 仅保留满足 $d(i,j) < r_i$ 的邻居。

4. **语义门控 (Feature Gating)：**
   - 在邻域构建阶段提前切断跨语义连接：
     $$\cos(f_i^{dino}, f_j^{dino}) > 0.6$$
   - 这能有效防止语义差异大的区域被错误连接。

5. **图对称化：** 若 $i \to j$ 或 $j \to i$ 任一方向的边存在，则保留无向边 $(i,j)$，提升图的连通稳定性。

6. **FAISS 加速 KNN 检索（推荐）：**
   - **构建索引：** 将所有有效高斯点的空间坐标传入 `faiss.IndexFlatL2`，构建精确 L2 索引。
   - **批量检索：** `distances, indices = index.search(means, k_max)` 一次性获取每个点的 $k_{max}$ 个最近邻及对应距离。
   - **向量化门控与裁剪：** 在 GPU 上以向量化操作同时执行特征门控（$\cos(f_i^{dino}, f_j^{dino}) > 0.6$）和自适应半径裁剪（$d(i,j) < r_i$），生成二值邻接掩码。
   - 此路径将复杂度从 chunked cdist 的 $O(N^2)$ 降至 $O(N \log N)$，且显存占用可控。

   ```python
   # 伪代码
   import faiss
   coords = cloud.means.cpu().numpy().astype(np.float32)
   index = faiss.IndexFlatL2(3)  # 3D 空间坐标
   index.add(coords)
   distances, indices = index.search(coords, k_max + 1)  # +1 因为包含自身
   ```

7. **纯 PyTorch 回退路径：** 当 FAISS 不可用或点数较少（< 5 万）时，回退到分块距离计算 (`chunked cdist`)，避免一次性全量距离矩阵带来的显存/内存峰值。块大小建议 2048–4096。

### 1.4 图区域生长 (BFS Graph Region Growing)

从种子点出发，通过广度优先搜索 (BFS) 将图划分为语义一致的连通区域。

1. **种子选择：** 按不透明度降序选择未访问点作为种子。

2. **生长接受条件（三个条件同时满足）：**
   - **特征相似度：** $\cos(h_i, h_j) > \tau_{feat}$（$\tau_{feat}$ 建议 0.7~0.85）
   - **法线一致性：** $|n_i \cdot n_j| > \tau_{normal}$（$\tau_{normal}$ 建议 0.6~0.8）
   - **空间约束：** $d(i,j) \le r_i \cdot \gamma$（$\gamma$ 建议 1.0~1.5）

   其中 $h_i$ 为 1.1 节中构建的混合特征向量。

3. **终止条件：** 当队列为空（无可接受的邻居）时，当前区域生长结束，选择下一个未访问种子点继续。

### 1.5 后处理：去噪与合并

初始分割可能存在碎片和噪声，需要后处理优化。

1. **标签传播 (Label Propagation)：**
   - 对小簇（点数 < `min_cluster_size`，建议 50~200）执行标签传播：将小簇中每个点的标签替换为其 K=10 近邻中出现最多的标签。
   - 这能有效修复孤立的误分点，减少碎片。

2. **KNN 投票平滑 (可选)：**
   - 对每个点检查其最近 K=10 个邻居。若超过 80% 的邻居为 Label A 而自身为 Label B，则更正为 A。
   - 能消除点云内部的椒盐噪声。

3. **连通分量分析 (Connected Components)：**
   - 对同一标签但空间上不连通的区域，拆分为不同的实例 ID。
   - 通过空间距离阈值（如 $2 \times$ 平均邻域距离）判断连通性。
   - 这解决了"两个相同语义但物理分离的物体被标为同一标签"的问题。

### 1.6 构建 3D 目标度量空间 $O_n$

对每个最终语义簇 $C_{3D,j}$，构建用于 FGW 匹配的度量测度空间。

1. **代表点采样：** 使用最远点采样 (FPS) 从簇中提取 $m$ 个代表点 $X = \{x_1, ..., x_m\}$（建议 $m = 50 \sim 200$）。

2. **图测地距离矩阵 $d_X$：**
   - 在代表点之间构建 KNN 图（$k = 5 \sim 10$）。
   - 计算最短路径距离作为测地距离近似。
   - *(可选加速)* 使用 Nyström 扩展从采样点子矩阵近似全量测地距离。

3. **概率测度 $\mu_X$：** 基于各代表点邻域内的不透明度之和归一化得到：
   $$\mu_X(i) = \frac{\sum_{p \in \mathcal{N}(x_i)} \alpha_p}{\sum_{j=1}^m \sum_{p \in \mathcal{N}(x_j)} \alpha_p}$$

4. **代表特征 $F^X$：** 对各代表点邻域内的 DINO 特征取加权平均。

5. **输出：** $O_n = (X, d_X, \mu_X, F^X)$，即一个带语义特征的 3D 度量测度空间。

## 🔷 阶段二：FGW 跨域拓扑对齐 (核心创新 · POT 加速)
**目标：** 为每个 3D 区域 $O_n$ 自动匹配最符合其结构逻辑的 2D 纹理 $M_k$（来自管线一阶段二的风格基元）。使用 POT 库的 GPU 加速 FGW 求解器，替代自实现 Sinkhorn 迭代。

### 2.1 FGW 问题定义

1. **输入：** 对每个 3D 区域 $O_n = (X, d_X, \mu_X, F^X)$ 和每个 2D 风格基元 $M_k = (Y, d_Y, \mu_Y, F^Y)$。

2. **联合匹配代价：**
   - **特征代价矩阵 $M$：** $M_{ij} = 1 - \cos(F_i^X, F_j^Y)$，衡量跨域语义距离。
   - **结构代价（内嵌于 FGW）：** 衡量内部拓扑的异构差异 $|d_X(i, k) - d_Y(j, l)|^2$。
   - **FGW 总目标：**
     $$\min_{\pi} \sum_{i,j} M_{ij} \pi_{ij} + \alpha \sum_{i,j,k,l} |d_X(i,k) - d_Y(j,l)|^2 \pi_{ij} \pi_{kl}$$

3. **关键超参 $\alpha$：**
   - $\alpha = 0$：纯特征匹配（等价于标准最优传输）。
   - $\alpha = 1$：纯结构匹配（等价于 Gromov-Wasserstein）。
   - 推荐 $\alpha = 0.5$：平衡语义与结构。

### 2.2 POT FGW 求解

直接调用 POT 的 GPU 加速 FGW 求解器，避免手动实现 Sinkhorn 迭代中的数值 instability：

```python
import ot

# M: 特征代价矩阵 (n_3d × n_2d), C1: 3D 内部距离矩阵, C2: 2D 内部距离矩阵
# p: 3D 概率测度, q: 2D 概率测度
pi = ot.gromov.fused_gromov_wasserstein(
    M=M,                              # 特征代价 (n_3d, n_2d)
    C1=d_X,                           # 3D 结构矩阵 (n_3d, n_3d)
    C2=d_Y,                           # 2D 结构矩阵 (n_2d, n_2d)
    p=mu_X,                           # 3D 概率测度 (n_3d,)
    q=mu_Y,                           # 2D 概率测度 (n_2d,)
    loss_fun='square_loss',           # L2 结构差异
    alpha=0.5,                         # 特征/结构平衡
    epsilon=0.01,                      # 熵正则化系数（越大匹配越模糊）
    max_iter=2000,
    tol=1e-9,
    verbose=False,
)
```

**POT 相比自实现的优势：**
- 内置对数域稳定化 (log-domain stabilization)，避免 $\exp(-\infty)$ 的数值溢出。
- 自动选择 Sinkhorn 迭代步长，无需手动调参。
- 支持 GPU 后端（`ot.gromov.fgw` 在 PyTorch 张量上运行，与已有 pipeline 无缝衔接）。

### 2.3 Coarse-to-Fine FGW（百万级点加速）

直接对全量代表点做 FGW 在百万级高斯场景下仍可能 OOM。沿用上文"POT 加速引擎详解"中的由粗到细策略：

1. **粗匹配：** 在粗聚类层（管线二 1.2）上以 Super-nodes 做 FGW → 得到粗粒度传输计划 $\pi_{coarse}$。
2. **Top-K 筛选：** 对每个 Super-node 保留 $\pi_{coarse}$ 中权重最高的 $K_{top}=3$ 个候选 2D 基元。
3. **细匹配：** 在各 Super-node 内部对 Top-K 候选做精细 FGW → 得到点级传输计划 $\pi_{fine}$。
4. **匈牙利指派：** 对 $\pi_{fine}$ 的每一行取 argmax，将对应的 2D 基元特征赋给 3D 高斯点簇，作为后续风格化的优化目标 $F_i$。

## 🔷 阶段三：多视角一致性风格生成 (NNFM)
**目标：** 冻结场景几何，通过 3D 优化实现无闪烁的风格迁移。

1. **隔离优化：** 仅更新球谐函数 (SH) 系数和基础颜色，完全冻结高斯点坐标 $xyz$ 和协方差矩阵参数。

2. **最近邻特征匹配 (NNFM Loss)：**
   - 多视角渲染当前场景，提取 VGG 特征 $\Phi(I_{render})$。
   - 最小化渲染特征与匹配到的目标纹理特征的格拉姆矩阵 (Gram Matrix) 差异。
   - **关键约束 (Stop-Gradient)：** 目标特征 $\Phi(M_k)$ 和高斯点内部的 $f_{dino}$ 必须 detach，防止梯度回传破坏几何场。

3. **法线感知一致性 (Normal-Aware Consistency)：**
   - 在损失函数中增加惩罚项：强制法线夹角极小（共面）的相邻高斯点保持相近的风格特征表达，从物理层面杜绝视点移动时的"纹理滑动 (Flickering)"。
