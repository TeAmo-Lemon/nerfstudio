# 🌟 系统架构设计书：SemGeo-FGW Splatting
**(面向 FGW 的结构感知 3D–2D 拓扑对齐 3DGS 风格化系统)**

## 核心思想

本系统将 3D 场景的局部区域与 2D 风格图的局部纹理，均抽象为**带有语义特征的度量测度空间 (Metric Measure Space)**。通过融合格罗莫夫-瓦瑟斯坦距离 (Fused Gromov-Wasserstein, FGW)，实现跨模态的纯结构拓扑对齐。

系统划分为两条管线：
- **管线一 (`dino_splatfacto`)：** 3DGS 语义特征蒸馏训练 + 2D 风格图纹理基元分解。产出带语义特征的 3DGS checkpoint 和风格基元。
- **管线二 (`semgeo_splatfacto`)：** 加载管线一的输出，完成聚类分解、FGW 拓扑对齐和最终风格生成。聚类放在训练后，因为高斯点数量在 densification/pruning 期间不断变化，聚类必须在固定高斯点集上进行。

# 🔵 管线一：DinoSplatfacto — 3D 特征蒸馏训练与风格基元分解
**目标：** 在 3DGS 重建的同时蒸馏 DINO 语义特征到每个高斯点；对 2D 风格图做纹理基元分解。产出带语义的 3DGS checkpoint 和风格基元。

**状态：✅ 已完成**

## 🔷 阶段一：3D 混合特征蒸馏训练

### 1.1 DINOv2 2D 特征预处理（训练前离线阶段）

使用 `extract_dino_features.py` 对所有训练视角图像离线提取 DINOv2 特征并降维。

1. **特征提取：** 使用 `dinov2_vitl14` 对所有训练图像提取 patch-level 特征（`x_norm_patchtokens`），每张图得到 $F_{2D} \in \mathbb{R}^{H \times W \times 1024}$。

2. **全局 PCA 降维：** 收集所有训练图像的 patch 特征向量，拟合全局 PCA，将维度从 1024 压缩至 **16 维**。8–16 维足以保留 DINOv2 的核心语义信息（主成分累积方差 > 80%）。

3. **特征图输出：** 将降维后的特征图 $F_{2D}^{low} \in \mathbb{R}^{H \times W \times 16}$ 保存为单个文件。

#### 特征文件格式 (`dino_features.pt`)

```python
{
    "version": 2,                              # 文件格式版本
    "feature_dim": 16,                         # PCA 降维后的特征维度
    "input_dir": "/path/to/image/root",        # 图像根目录
    "features": {
        "relative/path.jpg.pt": tensor(H, W, 16),   # 每张图一个 key，key 为相对路径
        "another/image.png.pt": tensor(H, W, 16),
        ...
    },
    "pca_state": {
        "components": tensor(effective_dim, 1024),   # PCA 主成分矩阵
        "mean": tensor(1024,),                       # PCA 均值向量
    },
}
```

**增量更新：** 提取脚本支持增量模式（`skip_existing=True`）——若 `dino_features.pt` 已存在且某张图的 key 已存在且维度匹配，则跳过该图；仅对新图像或维度不匹配的图像重新提取。新图像使用已有 `pca_state` 投影（不再重新拟合 PCA）。

**特征 key 命名规则：** `image_path.relative_to(image_root).with_suffix(".pt").as_posix()` ，例如 `images/DSC0001.jpg` → `"images/DSC0001.jpg.pt"`。

### 1.2 3DGS 模型扩展

修改 3DGS 底层数据结构，让每个高斯点同时携带颜色和语义信息。

1. **参数扩展：** 在 `DinoSplatfactoModel` 的 `gauss_params` 中增加 `dino_features` 参数，形状 `[N, 16]`，初始化为 `0.01 * randn(N, 16)`。

2. **双通道渲染（核心实现细节）：**
   - **Pass 1 (RGB)：** 标准 gsplat 渲染，使用 SH 系数（`features_dc` + `features_rest`），渲染模式 `"RGB+ED"`。
   - **Pass 2 (DINO 特征)：** 复用 gsplat rasterizer，将 `dino_features` 作为 `colors` 参数传入，`sh_degree=None`（不做 SH 变换），渲染模式 `"RGB"`。gsplat 的 alpha blending 对任意通道数均适用，因此 DINO 特征渲染等价于颜色渲染。
   - 两次 rasterization 共享同一组 `means/quats/scales/opacities/viewmat/K`。

3. **Densify/Prune 同步：** `DinoDefaultStrategy` 继承 gsplat 的 `DefaultStrategy`，重写 `_grow_gs` 和 `_prune_gs`，调用 `duplicate`/`split`/`remove` 时传入 `self.model_ref.gauss_params`（包含 `dino_features`），保证 DINO 特征与高斯点数量始终保持一致。每一步结束时会断言 `dino_features.shape[0] == means.shape[0]`。

### 1.3 特征蒸馏训练

在基础 3DGS 重建训练的同时进行特征蒸馏。

- **联合损失函数：** $Loss_{total} = Loss_{rgb} + \lambda_{feat} \cdot Loss_{feat}$，其中 $Loss_{feat} = \| F_{feat}^{render} - F_{2D}^{low} \|_1$

- **延迟启用策略：** DINO 损失在 `dino_loss_start_step`（默认 3000）步前不生效，让几何先行收敛。

- **超参设置：** $\lambda_{feat}$ 默认 0.05。

- **结果：** 训练完成后，每个高斯点携带一个 16 维 DINO 特征编码。输出为标准 3DGS checkpoint（`step-*.ckpt`）。

#### Checkpoint 保存格式

Checkpoint 中 `dino_features` 参数与标准 gsplat 参数并列保存：

```python
# state_dict 中的关键张量
{
    "means": tensor(N, 3),
    "scales": tensor(N, 3),
    "quats": tensor(N, 4),
    "features_dc": tensor(N, 1, 3),      # SH DC 分量 (RGB)
    "features_rest": tensor(N, 15, 3),    # SH 高阶分量
    "opacities": tensor(N, 1),
    "dino_features": tensor(N, 16),       # <-- 新增的 DINO 特征
}
```

`DinoSplatfactoModel.load_state_dict` 会在加载时自动将 `dino_features` 等 key 映射到 `gauss_params.*`，并处理 checkpoint 中高斯点数量与当前模型不一致的情况（如从不同阶段的 checkpoint 恢复）。

#### 训练数据流

```
extract_dino_features.py  →  dino_features.pt  (离线)
                                   ↓
DinoDatamanager.__init__  →  _ensure_dino_features()  (缺失时自动提取)
                                   ↓
DinoInputDataset.get_data(idx)  →  从 dino_features.pt 读取特征图
                                   →  放入 batch["dino_feature"]  (H, W, 16)
                                   ↓
DinoDatamanager.next_train(step)  →  将 dino_feature 移到 GPU
                                   →  返回 (camera, batch)
                                   ↓
DinoSplatfactoModel.get_loss_dict()
    ① rasterize RGB (with SH)     →  outputs["rgb"]
    ② rasterize DINO (no SH)      →  outputs["rendered_features"]
    ③ L1(rendered_features, batch["dino_feature"])  →  loss_dict["dino_feature_loss"]
```

## 🔷 阶段二：2D 风格图纹理基元分解
**目标：** 将单张风格图无监督地分解为一组局部纹理基元 prototype 向量，供管线二做 FGW 匹配使用。

**实现位置：** `DinoDatamanager._ensure_style_primitives()`（在 `datamanager.py` 中）

1. **Patch 级特征提取：** 对风格图（resize 到 560×560，padding 到 14 的倍数）提取 `dinov2_vitl14` 的 `x_norm_patchtokens`。

2. **L2 归一化 + K-Means 聚类：**
   - 对所有 patch 特征做 L2 归一化。
   - 使用纯 PyTorch 实现 K-Means（50 轮迭代），聚类为 K 个 prototype（建议 $K = 5 \sim 8$）。
   - 对聚类中心再做 L2 归一化，得到最终 prototype。

3. **输出文件格式 (`style_primitives.pt`)：**
   ```python
   {"prototypes": tensor(K, 1024)}   # L2 归一化后的 DINOv2 特征向量
   ```

4. **加载：** `DinoSplatfactoModel.__init__` 中将 `style_primitives` 注册为 `buffer`（`persistent=False`，不随 checkpoint 保存）。

# 🟢 管线二：SemGeoSplatfacto — 结构分解与 FGW 风格化
**目标：** 加载管线一的 3DGS checkpoint 和风格基元，对固定高斯点集进行聚类、构建度量空间、FGW 对齐和最终风格生成。

**状态：🔨 开发中（阶段一聚类进行中）**

## 🔷 阶段一：3D 结构感知分解与度量空间构建

加载管线一产出的 checkpoint，从固定高斯点集出发，完成语义聚类和度量空间构建。

### 1.1 构建增强混合特征空间 $V_{enhanced}$

将几何特征与语义特征融合，构建用于聚类的增强特征向量。

- **几何分量 $g_i$：** $[x, y, z,\; n_x, n_y, n_z,\; SH_0]$ —— 空间坐标 + 法线方向（从协方差矩阵最小特征向量提取）+ 基础颜色（SH 直流分量）
- **语义分量 $f_{dino,i}$：** 蒸馏后的 16 维 DINO 特征（从 checkpoint 读取）
- **标准化与白化：** 对几何和语义分量分别 Z-score 标准化后，拼接并白化 (Whitening) 处理，平衡几何与语义在距离度量中的贡献
- **权重：** 默认 $w_g = 1.0,\; w_d = 0.8$

### 1.2 粗粒度预聚类

当高斯点数 > 5 万时，先用 K-Means 做粗聚类（$K = 50 \sim 100$），将后续图计算限制在各粗簇内部，复杂度从 $O(N^2)$ 降至 $O(\sum_k N_k^2)$。簇边界点额外检索相邻簇的 KNN 以防止边界断裂。

**未来加速：** 可使用 FAISS GPU K-Means 替代纯 PyTorch 实现。

### 1.3 自适应邻域图构建

1. **有效点筛选：** $\sigma(\alpha) \ge 0.05$，按不透明度保留最多 `max_sample_points`（默认 200,000）个点。
2. **局部尺度估计：** $s_i = d_i^{(k_{base})}$，$k_{base} = 8$。
3. **自适应半径邻域：** $r_i = \alpha \cdot s_i$，$\alpha = 2.0$，取 $k_{max} = \max(knn\_k, 32)$ 个候选近邻后按半径裁剪。
4. **语义门控：** $\cos(f_i^{dino}, f_j^{dino}) > 0.6$，提前切断跨语义连接。
5. **图对称化：** 若 $i \to j$ 或 $j \to i$ 任一方向存在边，则保留无向边。
6. **当前实现：** 使用 chunked `torch.cdist` 计算距离，点数少时足够；**未来加速：** FAISS `IndexFlatL2` 可降至 $O(N \log N)$。

### 1.4 图区域生长 (BFS)

从种子点出发 BFS，三个接受条件同时满足：
- 特征相似度：$\cos(h_i, h_j) > \tau_{feat}$（0.7~0.85）
- 法线一致性：$|n_i \cdot n_j| > \tau_{normal}$（0.6~0.8）
- 空间约束：$d(i,j) \le r_i \cdot \gamma$（$\gamma = 1.0 \sim 1.5$）

### 1.5 后处理：去噪与合并

1. **标签传播：** 小簇（< `min_cluster_size`，建议 50~200）用 K=10 近邻众数替换标签。
2. **连通分量分析：** 同标签但空间不连通的区域拆分为不同实例 ID。

### 1.6 构建 3D 目标度量空间 $O_n$

对每个最终语义簇 $C_{3D,j}$：
1. **代表点采样：** FPS 提取 $m$ 个代表点（$m = 50 \sim 200$）。
2. **图测地距离矩阵 $d_X$：** 在代表点之间构建 KNN 图（$k = 5 \sim 10$），计算最短路径距离。
3. **概率测度 $\mu_X$：** 基于各代表点邻域内不透明度之和归一化。
4. **代表特征 $F^X$：** 邻域内 DINO 特征加权平均。
5. **输出：** $O_n = (X, d_X, \mu_X, F^X)$。

## 🔷 阶段二：FGW 跨域拓扑对齐 (待实现)
**目标：** 为每个 3D 区域 $O_n$ 自动匹配最符合其结构逻辑的 2D 纹理基元。

直接对代表点做 FGW 求解，推荐 $\alpha = 0.5$（平衡语义与结构），使用 POT `ot.gromov.fused_gromov_wasserstein` 作为求解器。

对于百万级点场景，采用 Coarse-to-Fine 策略：
1. 粗聚类层以 Super-nodes 做粗匹配
2. Top-K 候选筛选（$K_{top}=3$）
3. Super-node 内部细匹配
4. argmax 逐点指派

## 🔷 阶段三：多视角一致性风格生成 (待实现)
**目标：** 冻结场景几何，通过 3D 优化实现无闪烁的风格迁移。

1. **隔离优化：** 仅更新 SH 系数和基础颜色，冻结 $xyz$ 和协方差。
2. **NNFM Loss：** 最小化渲染特征与目标纹理特征的 Gram Matrix 差异。
3. **法线感知一致性：** 强制法线夹角极小的相邻高斯点保持相近风格特征表达。
